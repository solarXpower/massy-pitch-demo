Item Release Engine

From Upload to Live

Automated engine controlling supply, scarcity, and timing for every store on the platform.

1

Store Uploads Products

Store owner logs into BiziBid.com and bulk-uploads products — images, descriptions, starting bid prices. No limit on uploads to the pool.

BiziBid.com Portal
2

Weekly Selector — Monday 00:00 AST

BullMQ job randomly picks 25 items from the store's pending pool. Unsold items from the previous week are mixed back in, creating unpredictability and value.

Automated · BullMQ · Every Monday
3

Daily Releaser — 05:15 AST

Each day, 5 items are scheduled at random times between 05:30–15:00 AST. By Day 5, up to 25 items are live simultaneously — controlled scarcity.

Automated · Daily
4

Push Notification → Item Goes Live

Registered bidders get an FCM push notification when a new item drops. It appears in the store's keypad overlay and the Just Released strip on the home screen.

FCM Push · Socket.IO
5

Auction Runs 3 / 7 / 14 Days

Bidders compete in real-time. Outbid events fire instantly via Socket.IO rooms — one room per auction. The highest bidder when the timer ends wins.

Real-time · Socket.IO
6

Auction Closes → Win Key Generated

Server generates an 8-character alphanumeric code (e.g. HBD-X4K9). Winner is notified. Code is revealed in-app only — never sent in the push notification text.

Secure · Bcrypt-hashed in DB
